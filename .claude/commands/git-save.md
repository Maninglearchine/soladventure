현재 변경사항을 GitHub에 저장해라.

커밋 메시지는 사용자가 입력한 `$ARGUMENTS`를 사용한다.

작업 순서:
1. `git status`를 실행해서 변경된 파일을 확인한다.
2. 변경사항이 없으면 "변경사항 없음"이라고 알려주고 종료한다.
3. 현재 브랜치명을 확인한다.
4. `git add .`를 실행한다.
5. `git commit -m "$ARGUMENTS"`를 실행한다.
6. `git push`를 실행한다.
7. 마지막으로 아래 내용을 요약한다.
   - 현재 브랜치명
   - 커밋 메시지
   - 변경 파일 수
   - commit 성공 여부
   - push 성공 여부

주의사항:
- 커밋 메시지가 비어 있으면 실행하지 말고 커밋 메시지를 요청한다.
- push 중 오류가 발생하면 오류 내용을 요약하고 해결 방법을 제안한다.
